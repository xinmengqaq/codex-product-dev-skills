# Task BDD 块模板

每个 Task 前只放行为定向，不放工程设计。

```markdown
### Task N：<行为名称>

**输入依据**
- UI 原型：
- 设计文档：
- 当前代码事实：
- 用户明确要求：

**行为目标**
- 用户/玩家要完成：
- 系统必须产生：
- 用户/玩家必须感知到：

**正常路径**
- Given：
- When：
- Then：

**失败路径**
- 场景：
- 系统反馈：
- 可恢复方式：

**边界路径**
- 场景：
- 期望行为：

**必须遵守**
- 原型或设计中不能改变的布局、交互、状态流或体验目标。

**允许变通**
- 只允许在不改变行为目标和验收证据的前提下调整实现方式。

**禁止新增**
- 未被 UI 原型、设计文档、当前代码事实或用户明确要求支持的功能。

**验收证据**
- 浏览器 / 试玩 / 真实数据 / 真实模型调用 / 日志 / 截图 / 测试中至少列出可执行证据。
```

## 写法规则

- Task 名称必须是用户/玩家行为，不是技术模块名。
- `Given / When / Then` 可以用中文自然语言写，不需要 Cucumber 文件格式。
- `禁止新增` 必须具体列出 AI 可能会脑补的功能，不能只写“不要扩大范围”。
- `验收证据` 必须能被执行或观察，不能只写“实现完成”。
- 如果证据需要浏览器、试玩或真实模型调用，必须明确写出。

## 产品类示例

```markdown
### Task 1：用户能从原始素材生成可编辑结构化结果

**输入依据**
- UI 原型：`UI原型/01-import.html`
- 设计文档：`docs/specs/material-workbench.md`
- 当前代码事实：已有素材导入入口，但没有结构化编辑态
- 用户明确要求：第一版先走通从输入到编辑保存的闭环

**行为目标**
- 用户要完成：输入一段原始素材并得到结构化结果。
- 系统必须产生：可编辑的标题、分类、正文和标签字段。
- 用户必须感知到：系统提取了什么，哪些字段可以修改，保存是否成功。

**正常路径**
- Given：用户位于素材导入页面，输入一段原始素材。
- When：用户点击生成。
- Then：页面展示可编辑结构化结果，保存后刷新仍能看到。

**失败路径**
- 场景：输入为空或模型调用失败。
- 系统反馈：显示明确错误原因。
- 可恢复方式：用户能修改输入并重试。

**边界路径**
- 场景：输入很短、重复提交、刷新页面。
- 期望行为：不会生成假数据，不会重复保存脏数据，已保存内容可恢复。

**必须遵守**
- 保留原型中的输入区、结果编辑区、保存反馈和下一步入口。

**允许变通**
- 内部字段命名、组件拆分和接口结构由 writing-plans 根据代码库决定。

**禁止新增**
- 不新增素材市场、多人协作、批量导入、复杂权限和统计看板。

**验收证据**
- 浏览器真实输入一段素材，生成结果，编辑保存，刷新后结果仍存在。
- 如涉及模型，必须证明主路径不是 fixture 或硬编码输出。
```

## 游戏类示例

```markdown
### Task 1：玩家能完成一次摆摊成交或拒绝闭环

**输入依据**
- UI 原型：`prototype/stall-loop.html`
- 设计文档：`docs/design/stall-core-loop.md`
- 当前代码事实：已有背包和金币状态
- 用户明确要求：第一版先做可试玩核心循环

**行为目标**
- 玩家要完成：选择一个商品上架并开始营业。
- 系统必须产生：NPC 购买或拒绝的结果，库存和金币真实变化。
- 玩家必须感知到：NPC 为什么买或不买，下一步可以调价或换商品。

**正常路径**
- Given：玩家背包有商品并进入摊位界面。
- When：玩家上架商品并开始营业。
- Then：NPC 作出购买或拒绝决定，界面反馈原因，资源状态变化。

**失败路径**
- 场景：背包为空、商品价格无效、营业中重复上架。
- 系统反馈：给出可理解提示。
- 可恢复方式：玩家能返回背包、调价或取消操作。

**边界路径**
- 场景：连续多次营业、库存归零、金币变化刷新。
- 期望行为：状态不丢失，不出现负库存或重复结算。

**必须遵守**
- 保留原型中的商品区、摊位区、NPC 反馈区和资源栏。

**允许变通**
- NPC 决策算法可以先简单，但必须有可解释反馈。

**禁止新增**
- 不新增声望、每日结算、多摊位、复杂经济曲线、NPC 好感度。

**验收证据**
- 能实际试玩一次上架、营业、成交或拒绝、资源变化的闭环。
- 截图或录屏关键状态，必要时补充状态日志。
```
